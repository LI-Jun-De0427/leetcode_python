# LeetCode #3418: Maximum Amount of Money Robot Can Earn

這是一題非常有趣的 **動態規劃 (DP)** 進階題。它在傳統的「路徑最值問題」上，增加了一個「有限次數的能力」限制。

---

## 題目敘述
機器人從 $m \times n$ 網格的左上角 $(0, 0)$ 出發，目標是右下角 $(m-1, n-1)$。
* **移動規則：** 只能向 **右** 或 向 **下**。
* **格子內容：** * 正數：獲得金幣。
    * 負數：遇到強盜，會被搶走該金額。
* **特殊能力：** 機器人一路上最多可以 **抵消 (Neutralize) 2 次** 強盜（讓該格的負數變成 0）。
* **目標：** 走到終點時能獲得的最大金幣數。

---

## 解題思路：三維動態規劃 (3D DP)

通常路徑問題只需要一個 2D 表格，但因為這題有「抵消次數」的限制，我們需要多一個維度來紀錄「用了幾次能力」。

### 1. 狀態定義
我們定義 `dp[r][c][k]` 為走到位置 `(r, c)` 且已經使用了 `k` 次抵消能力時的最大金額。
* `k = 0`: 從未用過能力。
* `k = 1`: 已經用過 1 次能力。
* `k = 2`: 已經用過 2 次能力。

### 2. 轉移邏輯 (核心思維)
每一格的結果都來自「左方」或「上方」：
1. **不使用能力：** 目前的錢 = `max(左方, 上方)` + `當前格子的值`。
2. **使用能力 (僅限當前格子為負數時)：**
   目前的錢 = `max(左方已用 k-1 次, 上方已用 k-1 次)` + `0`。

---

## 形象化理解：三層地圖
你可以想像這題有 **三張一模一樣的地圖** 疊在一起：
* **第 0 層**：完全遵守規則，遇到強盜就扣錢。
* **第 1 層**：你獲得了一次免死金牌，可以從第 0 層「跳」過來，並在該格不扣錢。
* **第 2 層**：你獲得了第二次免死金牌，可以從第 1 層「跳」過來。

我們的目標就是看這三張地圖的右下角，哪一個數字最大！
