# Q3. Minimum Operations to Transform Array into Alternating Prime (轉換交替質數陣列的最小操作)

## 題目原文 (Problem Description)
> You are given an integer array `nums`. An array is considered **alternating prime** if:
> - Elements at **even** indices (0, 2, 4...) are **prime** numbers.
> - Elements at **odd** indices (1, 3, 5...) are **non-prime** numbers.
>
> In one operation, you may **increment** any element by 1.
> **Return the minimum number of operations required to transform `nums` into an alternating prime array.**

---

## 題目在說什麼？

這題要求我們把一個陣列變成「交替排列」的格式：
1. **偶數位置**（第 0, 2, 4... 格）：必須要是**質數**。
2. **奇數位置**（第 1, 3, 5... 格）：必須要是**非質數**。

**規則**：你只能「往上加 1」。
* 如果那一格應該是質數，但現在不是，你要一直加到它是質數為止。
* 如果那一格應該是非質數，但現在是質數，你要往上加到它變成非質數為止。
* 最後算出你總共「按了幾次 +1」。

---

## 解題想法 (Step-by-Step)

這題有兩個核心挑戰：如何快速判斷質數，以及如何找到下一個符合條件的數。

1. **預處理質數表**：
   因為陣列中的數字最大到 $10^5$，且我們可能需要往上加到下一個質數，所以我們預先計算出 $10^5$ 左右的所有質數。我們使用 **「埃拉托斯特尼篩法 (Sieve of Eratosthenes)」**，這是一種超級快的方法來找出範圍內的所有質數。

2. **掃描陣列**：
   用一個迴圈走過整個陣列，根據位置是「奇數」還是「偶數」來執行不同邏輯。

3. **計算步數**：
   * **偶數位置（要質數）**：如果目前的數不是質數，就往上找第一個出現的質數。
   * **奇數位置（要非質數）**：如果目前的數是質數，就直接加 1（因為質數加 1 之後，除了 2 以外一定是偶數，而偶數一定是合數/非質數；如果是 2 加 1 變成 3，3 還是質數，就再加 1 變成 4）。簡單來說，就是加到它不是質數為止。

## 📚 新手筆記 (Key Takeaways)

### 1. 質數篩法 (Sieve of Eratosthenes)
如果你一格一格去判斷一個大數字是不是質數，速度會很慢。
* **比喻**：篩法就像拿一個篩子，把 2 的倍數、3 的倍數...通通抖掉，剩下的就是質數。
* **優點**：這在處理大量數字（例如本題到 $10^5$）的質數判定時，效率非常高。

### 2. `bisect_left` 是什麼？
這是一個「二分搜尋」工具。
* **情境**：當你有一個排好序的清單（例如我們的質數表），你想快速知道「某個數字應該插在哪裡」，或「比這個數大的下一個質數在哪」。
* **優點**：用這個工具比你從頭開始一個一個找（線性搜尋）要快得多。

### 3. 非質數的定義
題目提醒我們「質數是大於 1 的自然數」。
* **重點**：這代表 **0 和 1 也是非質數**。
* **應用**：在奇數位置（需要非質數），如果數字本來就是 1，我們不需要做任何加法操作，因為它已經符合條件了。

---

### 4. 複雜度分析 (Complexity Analysis)

為了確保公式能正確顯示，請注意符號前後的空格：

* **時間複雜度**： $O(M \log \log M + N \log P)$
    * **M**： 預處理質數的範圍（約 $10^5$）。
    * **N**： 陣列的長度。
    * **P**： 範圍內的質數總數。
    * **解釋**： 預處理質數表消耗 $O(M \log \log M)$，而對陣列中每個數字進行二分搜尋消耗 $O(N \log P)$。

* **空間複雜度**： $O(M)$
    * **解釋**： 用於儲存長度為 $M$ 的質數標記表（布林陣列）。
