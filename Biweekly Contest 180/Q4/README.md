# Q4. Maximum Value of Concatenated Binary Segments (拼接片段的最大值)

## 題目原文 (Problem Description)
> 有兩個陣列 `nums1` 和 `nums0`，分別代表第 `i` 個片段中 `1` 的數量與 `0` 的數量。
> 每個片段的構造方式為：**先放 `nums1[i]` 個 '1'，後面再接 `nums0[i]` 個 '0'**。
> 你可以自由調整這些片段的順序。
>
> **請回傳重新排列並拼接後，所能形成的二進位字串之最大十進位值（對 10⁹ + 7 取餘數）。**

---

## 題目在說什麼？

想像你有很多塊積木，每塊積木都是「前半段全是 1，後半段全是 0」。
* 片段範例：`nums1=2, nums0=1` -> 形成 `"110"`。
* 你的任務：像拼字遊戲一樣，把這些積木排成一列。
* 目的：拼出來的二進位數字越**大**越好。
* **直覺**：為了讓數字大，我們要把 `1` 盡量往左邊（高位）推，把 `0` 往右邊（低位）推。

---

## 解題想法 (Step-by-Step)

### 1. 片段分類與處理
為了效率，我們可以把積木分成三類：
* **純 1 積木**：最強大的積木，絕對要放在最左邊（最高位）。
* **純 0 積木**：最弱的積木，絕對要放在最右邊（最低位）。
* **混和積木 (有 1 有 0)**：最糾結的部分，需要特殊的排序規則。

### 2. 混和積木的排序規則 (Greedy Strategy)
對於混和積木，我們要決定 A 放 B 前面好，還是 B 放 A 前面好？
* 規則是：**1 越多且 0 越少的積木，權力越大，越要往前排。**
* 你的程式碼使用了 `(-x[0], x[1])` 作為 key，意思是先比 1 的數量（多者優先），如果 1 一樣多，再比 0 的數量（少者優先）。

### 3. 計算總值 (Modular Arithmetic)
因為拼接出來的數字會非常巨大，我們使用「預處理 2 的次方表」來快速計算位移後的數值：
* **位移公式**：把原本的數字左移 $k$ 位，等同於 `v * (2^k)`。
* **填入 1**：把 $k$ 個 `1` 填入低位，公式為 `(v * 2^k) + (2^k - 1)`。

---

## 新手筆記 (Key Takeaways)

### 1. 預處理 (Pre-computation)
當我們處理大數據（ $10^5$ 量級）且需要頻繁用到 $2^k$ 時，先用一個陣列 `p` 把所有結果算好存起來。
* **優點**：這能把時間從「每次計算都花 $O(\log k)$」大幅降到「每次查表只需 $O(1)$」。
* **應用**：這在需要大量取模（Modulo）運算的題目中非常實用。

### 2. 位移與數值的關係
在二進位中，後方每多出一個 `0`，就代表原來的數值乘以 `2`。
* **範例**： `11` (十進位的 3) 變成 `110` (十進位的 6) $\rightarrow$ 等於乘以 $2^1$。
* **程式實作**： `v = (v * p[b]) % M`。這行代碼實際上是將目前的數值 `v` 向左移動 `b` 位。

---

### 3. 複雜度分析 (Complexity Analysis)

* **時間複雜度**： $O(L + N \log N)$
    * **L**： 總位元長度（用於預處理次方表）。
    * **N**： 片段數量（用於排序混和片段）。
    * **解釋**： 預處理次方表花費 $O(L)$，而對片段進行排序花費 $O(N \log N)$。

* **空間複雜度**： $O(L)$
    * **解釋**： 主要是用來存儲長度為 $L$ 的次方表 `p`。
